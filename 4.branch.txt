## 브랜치?
작업이 직선으로 이어지지 않고, 곁가지로 뻗어나갈 수 있게 해 주는 도구
ex) 안정화 버전을 유지하고 개발용 브랜치를 만들어 작업후, 완성이 되면
안정화 버전에 반영한다.

브랜치를 사용하면 서로 다른작업을 진행하더라도 병합하기 수월해진다.
협업에도 좋지만 혼자 놀기에도 좋은 git의 핵심기능

git에서 브랜치는 매우 가볍게 동작하므로 하루에 수백개씩 만들었다 지워도
부담이 없다. 원래 그렇게 쓰라고 있는 기능이다.
사소한 작업이라도 일단 브랜치부터 만들고 보는 습관을 들이는 것이 좋다.

## 브랜치 목록 보기
>> git branch

## 브랜치 생성하기
>> git branch <branch_name>

## 브랜치 삭제하기
>> git branch -d <branch_name>
